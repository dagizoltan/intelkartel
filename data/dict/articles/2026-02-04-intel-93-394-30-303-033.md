<h2 class="wp-block-heading">TECHNICAL ANNEX</h2>



<p class="wp-block-paragraph"><strong>Classification:</strong> CONFIDENTIAL / CYBER‑SIGINT<br><strong>Annex ID:</strong> JTF‑CYB‑INT‑0276‑A</p>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h2 class="wp-block-heading">1. NETWORK‑LEVEL ATTACK VECTORS</h2>



<h3 class="wp-block-heading">1.1 Rogue Base Station Operations</h3>



<p class="wp-block-paragraph">Analysis confirms deployment of <strong>unauthorized base transceiver stations (BTS)</strong> mimicking legitimate carrier infrastructure.</p>



<p class="wp-block-paragraph"><strong>Observed capabilities:</strong></p>



<ul class="wp-block-list">
<li>Broadcast of spoofed MCC/MNC identifiers</li>



<li>Forced handset reselection via higher signal strength</li>



<li>Downgrade attacks (LTE/5G → GSM) to remove encryption</li>
</ul>



<p class="wp-block-paragraph"><strong>Indicators:</strong></p>



<ul class="wp-block-list">
<li>Sudden LAC/CID changes without geographic movement</li>



<li>Abnormal Timing Advance (TA) values</li>



<li>Ciphering disabled or A5/0 fallback</li>
</ul>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h3 class="wp-block-heading">1.2 IMSI &amp; TMSI Harvesting</h3>



<p class="wp-block-paragraph">Captured logs indicate systematic collection of subscriber identifiers.</p>



<p class="wp-block-paragraph"><strong>Methods:</strong></p>



<ul class="wp-block-list">
<li>Silent SMS paging</li>



<li>Location update requests</li>



<li>Attach/detach manipulation</li>
</ul>



<p class="wp-block-paragraph"><strong>Data collected:</strong></p>



<ul class="wp-block-list">
<li>IMSI</li>



<li>IMEI</li>



<li>MSISDN (correlated post‑collection)</li>



<li>Mobility patterns</li>
</ul>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h2 class="wp-block-heading">2. TELECOM INFRASTRUCTURE MANIPULATION</h2>



<h3 class="wp-block-heading">2.1 Signaling Exploitation</h3>



<p class="wp-block-paragraph">Evidence suggests abuse of legacy signaling protocols.</p>



<p class="wp-block-paragraph"><strong>Suspected vectors:</strong></p>



<ul class="wp-block-list">
<li>SS7 MAP requests (AnyTimeInterrogation, ProvideSubscriberInfo)</li>



<li>Diameter misconfiguration exploitation</li>



<li>Inter‑carrier trust abuse</li>
</ul>



<p class="wp-block-paragraph"><strong>Effects:</strong></p>



<ul class="wp-block-list">
<li>Call and SMS redirection</li>



<li>Location tracking</li>



<li>Call forwarding activation without subscriber awareness</li>
</ul>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h3 class="wp-block-heading">2.2 SIM &amp; Authentication Attacks</h3>



<p class="wp-block-paragraph"><strong>Observed tactics:</strong></p>



<ul class="wp-block-list">
<li>SIM swap facilitation via social engineering and compromised retail access</li>



<li>Ki extraction attempts using downgraded cipher modes</li>



<li>OTA (Over‑The‑Air) message injection</li>
</ul>



<p class="wp-block-paragraph"><strong>Artifacts recovered:</strong></p>



<ul class="wp-block-list">
<li>Modified SIM toolkits</li>



<li>Non‑standard OTA headers</li>



<li>Replayable authentication sequences</li>
</ul>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h2 class="wp-block-heading">3. DEVICE‑LEVEL COMPROMISE</h2>



<h3 class="wp-block-heading">3.1 Firmware &amp; Baseband Attacks</h3>



<p class="wp-block-paragraph">Forensic analysis identified tampered baseband firmware on seized devices.</p>



<p class="wp-block-paragraph"><strong>Characteristics:</strong></p>



<ul class="wp-block-list">
<li>Disabled user notification flags</li>



<li>Hidden diagnostic interfaces enabled</li>



<li>Persistent monitoring modules surviving factory reset</li>
</ul>



<p class="wp-block-paragraph"><strong>Risk:</strong><br>Baseband compromise bypasses OS‑level security and standard mobile antivirus detection.</p>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h3 class="wp-block-heading">3.2 Payload Delivery</h3>



<p class="wp-block-paragraph">Delivery vectors include:</p>



<ul class="wp-block-list">
<li>Malicious configuration profiles</li>



<li>Zero‑click signaling payloads</li>



<li>Compromised charging accessories (USB‑C inline implants)</li>
</ul>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h2 class="wp-block-heading">4. COMMAND &amp; CONTROL (C2)</h2>



<h3 class="wp-block-heading">4.1 Communications Architecture</h3>



<p class="wp-block-paragraph">C2 traffic exhibits:</p>



<ul class="wp-block-list">
<li>Short‑burst encrypted transmissions</li>



<li>Domain fronting</li>



<li>Fast‑flux DNS rotation</li>
</ul>



<p class="wp-block-paragraph"><strong>Protocols observed:</strong></p>



<ul class="wp-block-list">
<li>Custom TLS over TCP/443</li>



<li>Encrypted UDP tunnels</li>



<li>Opportunistic Bluetooth mesh relays</li>
</ul>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h3 class="wp-block-heading">4.2 Traffic Signatures</h3>



<p class="wp-block-paragraph">Packet inspection revealed:</p>



<ul class="wp-block-list">
<li>Non‑standard cipher suites</li>



<li>Repeated session renegotiation</li>



<li>Metadata‑heavy, low‑content payloads</li>
</ul>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h2 class="wp-block-heading">5. MEDIA ACQUISITION &amp; COERCIVE USE</h2>



<h3 class="wp-block-heading">5.1 Digital Asset Handling</h3>



<p class="wp-block-paragraph">The network prioritizes acquisition of sensitive media to apply leverage.</p>



<p class="wp-block-paragraph"><strong>Technical indicators:</strong></p>



<ul class="wp-block-list">
<li>Automated cloud scraping</li>



<li>Account token replay</li>



<li>Metadata stripping and re‑encoding</li>
</ul>



<p class="wp-block-paragraph"><strong>Processing pipeline:</strong></p>



<ol class="wp-block-list">
<li>Acquisition</li>



<li>Sanitization</li>



<li>Selective editing</li>



<li>Encrypted distribution</li>
</ol>



<hr class="wp-block-separator has-alpha-channel-opacity">



<h2 class="wp-block-heading">6. FORENSIC EVIDENCE SNAPSHOT</h2>



<h3 class="wp-block-heading">6.1 Log Artifacts</h3>



<ul class="wp-block-list">
<li>Forced network reselection timestamps</li>



<li>Authentication failures followed by downgrade success</li>



<li>Repeated silent SMS delivery confirmations</li>
</ul>



<h3 class="wp-block-heading">6.2 Hardware Evidence</h3>



<ul class="wp-block-list">
<li>FPGA‑based radio modules</li>



<li>Reflashed SDR units</li>



<li>Battery‑powered portable BTS equipment</li>
</ul>

<h2 class="wp-block-heading">7. DETECTION &amp; COUNTER‑SIGINT MEASURES</h2>



<h3 class="wp-block-heading">7.1 Detection</h3>



<ul class="wp-block-list">
<li>Continuous RF spectrum monitoring</li>



<li>Cell ID consistency validation</li>



<li>Subscriber anomaly clustering</li>
</ul>



<h3 class="wp-block-heading">7.2 Mitigation</h3>



<ul class="wp-block-list">
<li>Enforce LTE/5G‑only modes where possible</li>



<li>Disable legacy GSM support</li>



<li>Deploy baseband integrity checks</li>



<li>Harden inter‑carrier signaling firewalls</li>
</ul>

<p class="wp-block-paragraph">The technical sophistication observed indicates:</p>



<ul class="wp-block-list">
<li>Experienced operators</li>



<li>Access to specialized hardware</li>



<li>Knowledge of telecom standards and legacy weaknesses</li>
</ul>



<p class="wp-block-paragraph">The threat is <strong>persistent</strong>, <strong>mobile</strong>, and <strong>difficult to attribute</strong>, aligning with advanced criminal or hybrid‑warfare cyber actors.</p>
